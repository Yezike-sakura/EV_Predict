# 创新规约

## 研究方向

当前计划中的模型方向是一个面向 EV 充电预测的混合时空模型，包含：

- 残差混合动态图生成
- 基于 Patch 的时空耦合
- 节点级冷启动归纳评估

## 创新点一：残差混合图

目标：

- 将静态图结构和动态图残差结合
- 用多模态先验特征生成动态空间关系
- 提升模型稳定性和冷启动泛化能力

实现约束：

- 静态图底座来自已有邻接矩阵或距离先验
- 动态图生成要尽量轻量
- 避免粗暴的 `O(N^2)` 全连接重构
- 更适合采用节点嵌入交互或注意力方式

预期输出：

- 动态或融合图张量形状为 `[B, P, N, N]`

## 创新点二：Patch 级时空耦合

目标：

- 把长序列切成 patch
- 在 patch 级别做空间聚合
- 在 patch 之间做时间注意力

预期张量流：

1. 输入张量为 `[B, C, N, L]`
2. 重塑为 `[B, C, N, P, P_len]`
3. 用融合图做 patch 内空间融合
4. 在 `P` 维做跨 patch 的时间建模

## 创新点三：冷启动归纳设置

目标：

- 在输入构造阶段屏蔽一部分节点
- 外部特征保持可用
- 训练时仅对老站点回传梯度
- 测试时仅在被屏蔽节点上评估

Phase 1 预期改动：

- `code/parse.py`：新增 `cold_start` 和 `cold_ratio`
- `code/utils.py`：生成 mask 并对历史输入致盲
- `code/train.py`：loss 仅作用于老站点，指标仅统计新站点

## 当前实现顺序

1. 先完成 cold-start 机制
2. 验证训练和评估逻辑
3. 再创建 `code/models/Proposed/` 并开始原创模型骨架
